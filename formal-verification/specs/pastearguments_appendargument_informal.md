## Purpose

`AppendArgument` appends a single string argument to a `StringBuilder` that is accumulating a Windows command-line string, quoting and escaping the argument as necessary.

The goal is a **round-trip guarantee**: the emitted text, when parsed by the Win32 `CommandLineToArgvW` function (or the post-2008 dotnet parser for non-argv[0] tokens), yields back the original argument string unchanged.

## Parsing Rules (Windows CommandLineToArgvW)

The escaping rules implemented here are the standard Windows command-line parsing rules for non-`argv[0]` tokens:

1. **Backslash is literal** unless followed by a double-quote.
2. **2N backslashes + `"`** → N literal backslashes + start/end of quoted region.
3. **2N+1 backslashes + `"`** → N literal backslashes + literal `"` character.
4. **Whitespace outside a quoted region** ends the current token.
5. **(Post-2008)** A closing `"` immediately followed by another `"` → literal `"`, and parsing remains in quoted mode.

## Internal Escaping Rules (within quoted region)

When the argument must be quoted (`Q4`), the escaping applied character-by-character is:

| Input pattern | Output |
|---------------|--------|
| N backslashes at end of argument | `2N` backslashes (doubled) |
| N backslashes followed by `"` | `2N+1` backslashes followed by `"` |
| N backslashes not at end and not before `"` | N backslashes (unchanged) |
| Single `"` | `\"` |
| Any other character | unchanged |

## Inferred Design Intent

The function is a faithful implementation of the Win32 quoting specification documented in MSDN and the "layered quoting" appendix of the C runtime source. The design intent is:

1. Avoid quoting when unnecessary (simple fast path for common case).
2. Always produce a token that unambiguously round-trips via `CommandLineToArgvW`.
3. Handle the `2N` and `2N+1` backslash rules correctly, which are the primary source of bugs in naïve implementations.
4. The separator space (Q1) integrates the argument into a complete command line built up across multiple calls.

| OQ2 | **Adjacent quoted strings**: The comment mentions the post-2008 rule "closing quote followed by another quote → literal quote". This function does not emit adjacent quoted strings, but does the round-trip guarantee hold for a concatenation of multiple calls if a token was already in a quoted region? |
| OQ3 | **Null handling**: the method does not null-check `argument`; if `null` is passed, `argument.Length` will throw a `NullReferenceException`. Should the precondition P2 be enforced with a guard? |

## Properties to Formalize in Lean

1. `∀ s, ContainsNoWhitespaceOrQuotes(s) = true → AppendArgument("", s) = s`
2. `AppendArgument("", "") = "\"\""`
3. `∀ sb, argument, sb.Length > 0 → (AppendArgument sb argument).first_char = ' '`
4. **Round-trip** (primary): `∀ s, commandLineToArgv(AppendArgument("", s)) = [s]`
5. **Backslash doubling at end**: If `argument` ends with `k` backslashes and is otherwise quotable, the output contains `2k` backslashes before the closing `"`.
6. **Backslash rule before internal quote**: If `argument` contains `k` backslashes followed by `"`, the output contains `2k+1` backslashes followed by `"`.
7. **Monotone length**: `AppendArgument("", s).Length ≥ s.Length` (quoting never makes a token shorter).
